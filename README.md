  # Лабораторная работа №2: Обработка массивов
  
  Реализация программы для выполнения арифметических операций над массивом целых чисел на основе управляющего кода.
  
  ## Описание задачи
  
  Программа принимает на вход управляющее число (код операции) и массив целых чисел. В зависимости от кода, выполняется одно из четырех действий.
  
  ### Параметры ввода:
  
  - **Управляющее число:** `0`, `1`, `2` или `3`.
  - **Массив:** до 100 целых чисел, разделенных пробелом.
  - **Завершение строки:** символ переноса строки (`\n`).
  
  ---
  
  ## Функциональные возможности
  
  Программа обрабатывает данные по следующей логике:
  
  |  Код  | Операция  | Описание                                                                       |
  | :---: | :-------- | :----------------------------------------------------------------------------- |
  | **0** | `abs_max` | Находит максимальное по модулю число в массиве.                                |
  | **1** | `abs_min` | Находит минимальное по модулю число в массиве.                                 |
  | **2** | `diff`    | Вычисляет разницу между максимальным и минимальным по модулю элементами.       |
  | **3** | `sum`     | Находит сумму элементов, начиная с максимального по модулю и до конца массива. |

  # Лабораторная работа №3: Форматирование и фильтрация текста
  
  Программа для автоматической обработки текстовых данных, их структурирования и фильтрации по заданным критериям.
  
  ## Описание задачи
  
  Программа считывает поток текста и выполняет его преобразование. Чтение данных продолжается до тех пор, пока не встретится предложение-терминатор: **"Dragon flew away!"**.
  
  ### Правила форматирования:
  
  1. **Разделение:** Каждое новое предложение выводится с новой строки.
  2. **Очистка:** Удаляются символы табуляции в начале предложений.
  3. **Фильтрация:** Предложения, содержащие цифру **7** (в любом месте, даже внутри слова), полностью удаляются из вывода.
  
  ## Функциональные возможности
  
  Логика обработки предложений базируется на символах пунктуации и финальной статистике:
  
  | Элемент         | Описание                                                                         |
  | :-------------- | :------------------------------------------------------------------------------- |
  | **Разделители** | Предложением считается текст, заканчивающийся на `.`, `;` или `?`.               |
  | **Терминатор**  | Строка `Dragon flew away!` завершает работу программы.                           |
  | **Исключение**  | Любое предложение с цифрой `7` (например, `agent007` или `7 дней`) игнорируется. |
  | **Статистика**  | В финале выводится количество предложений до и после фильтрации.                 |

  # Лабораторная работа №4: Сравнение алгоритмов сортировки
  
  Программа для анализа производительности различных алгоритмов сортировки на массиве из 1000 целых чисел.
  
  ## Описание задачи
  
  Программа принимает на вход массив целых чисел и выполняет его сортировку двумя способами, замеряя время выполнения каждого алгоритма с помощью функций стандартной библиотеки.
  
  ## Функциональные возможности
  
  Программа реализует два подхода к упорядочиванию данных для сравнения их эффективности:
  
  | Метод сортировки         |  Реализация   | Описание                                                                  |
  | :----------------------- | :-----------: | :------------------------------------------------------------------------ |
  | **Сортировка пузырьком** | `bubble_sort` | Классический алгоритм со сложностью $O(n^2)$.                             |
  | **Быстрая сортировка**   |    `qsort`    | Использование стандартной функции библиотеки со сложностью $O(n \log n)$. |
  | **Замер времени**        | `time/clock`  | Подсчет времени работы каждого алгоритма в микросекундах или тиках.       |

  # Курсовая работа: Система обработки текстовых данных
  
  Консольное приложение на языке C для высокоуровневой обработки динамических текстовых структур с использованием стандартных библиотек и ANSI-управления цветом.
  
  ## Используемые функции
  
  В рамках данной работы реализованы:
  
  - **Динамическое управление памятью:** работа с неограниченными по длине строками и массивами строк.
  - **Парсинг текста:** выделение предложений по заданным разделителям и токенизация слов.
  - **Алгоритмическая обработка:** фильтрация, сортировка через `qsort` и преобразование подстрок.
  - **Интерфейс командной строки:** обработка кодов команд и вывод согласно спецификации.
  
  ## Функциональные возможности
  
  Программа выполняет первичную обработку (удаление дубликатов без учета регистра) и предоставляет 5 основных функций:
  
  |  Код  | Функция           | Описание                                                     |
  | :---: | :---------------- | :----------------------------------------------------------- |
  | **0** | `Default`         | Вывод текста после удаления дубликатов.                      |
  | **1** | `Blue Highlight`  | Вывод слов после "define BLUE" голубым цветом (ANSI).        |
  | **2** | `RLE-Compression` | Замена цепочек одинаковых букв (напр. "AaaA" → "4A").        |
  | **3** | `Filter Words`    | Удаление предложений, где количество слов кратно 3.          |
  | **4** | `Smart Sort`      | Сортировка по убыванию суммы длин первого и последнего слов. |
  | **5** | `Help`            | Вывод справки по доступным командам.                         |

  # Лабораторная работа №1: Regex File Link Parser

  ## Описание проекта
  Программа для автоматизированного поиска и анализа ссылок на файлы в текстовых массивах. Использует регулярные выражения для извлечения             структурированной информации из неструктурированного ввода.
    
  ## Технические задачи
  * **Поиск ссылок:** Распознавание URL с опциональными протоколами (`http`, `ftp`, `mailto` и т.д.) и префиксом `www`.
  * **Выделение компонентов:** Точное извлечение доменного имени сайта и финального имени файла с его расширением.
  * **Потоковая обработка:** Чтение данных до терминатора предложения `Fin.`.
  * **Валидация:** Гарантированная работа с глубокими путями на сервере (директориями).
    
  ## Логика работы (RegEx)
  
  Для решения задачи используется паттерн, учитывающий вариативность протоколов и путей.

  Регулярное выражение, использованное в работе:
  ```regex
  (?:[a-zA-Z]+:\/\/)?(?:www\.)?([a-zA-Z0-9.\-]+\.[a-z]{2,})\/(?:[^\/\s]+\/)*([^\/\s]+\.[a-zA-Z0-9]+)
  ```

  # Лабораторная работа №2: Unrolled Linked List API
  
  Реализация высокоэффективной структуры данных — **развёрнутого связного списка** на языке C.

  
  ## О структуре данных
  **Unrolled Linked List** (Развёрнутый список) объединяет преимущества массивов и связных списков. Каждый узел (блок) содержит массив из нескольких элементов (`BLOCK_SIZE = 4`), текущий счетчик элементов и указатель на следующий узел.
  
  
  ### Преимущества реализации:
  * **Cache Locality:** Элементы внутри одного блока расположены в памяти последовательно, что ускоряет доступ.
  * **Снижение оверхеда:** Меньшее количество указателей по сравнению с классическим связным списком.
  * **Плотная упаковка:** Гарантируется максимальное заполнение всех блоков (кроме последнего), что предотвращает фрагментацию данных.
  
  ## Реализованный функционал (API)
  
  ### Управление жизненным циклом
  * `list_init()` — создание и инициализация структуры.
  * `list_destroy()` — корректная очистка всей выделенной памяти.
  
  ### Модификация данных
  * `list_push_front` / `list_push_back` — добавление элементов в начало и конец.
  * `list_push(index)` — вставка по глобальному индексу с сохранением плотности структуры.
  * `list_pop(index)` — удаление элемента с последующим перераспределением (сдвигом) данных между блоками для исключения пустот.
  
  ### Алгоритмическая обработка
  * **Сортировка:** Упорядочивание всех элементов с сохранением плотной упаковки блоков.
  * **Дедупликация:** Удаление повторяющихся значений за один проход.
  * **Трансформация:** Разворот порядка блоков (`reverse_blocks`) без изменения порядка элементов внутри них.
  * **Статистика:** Вычисление медианы списка и подсчет общего количества элементов.
  
  ## Технические особенности реализации
  
  1. **Балансировка блоков:** При любой вставке или удалении алгоритм автоматически перераспределяет элементы. Если блок переполняется, создается новый; если после удаления блок пустеет — он немедленно освобождается.
  2. **Глобальная индексация:** Реализована логика отображения «глобального индекса» (от 0 до N) в пару «указатель на блок + индекс в массиве».
  3. **Безопасность памяти:** Работа ведется с учетом динамической аллокации, исключая утечки памяти при сложных операциях (например, при удалении дубликатов или сортировке).
